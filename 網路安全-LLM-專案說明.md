### 第二步：核心技術實作指南
要在期末報告拿到「傑出 (O)」評級，必須證明你們不是單純把文字丟給 ChatGPT，而是具備嚴謹的資安工程思維。

#### 1. 意圖分析 (Intent Analysis) 的設計
現代的釣魚攻擊已經從單純的「惡意連結」進化為「武器化的文本 (weaponized text)」，利用流暢的對話進行社交工程。系統的目標不是找尋錯字，而是識別駭客的**意圖**。
你們需要在後端定義好幾個「危險意圖標籤 (Threat Intents)」：

- `Urgency` (急迫感：例如「您的帳戶將在 24 小時內凍結」)
- `Authority Impersonation` (權威冒充：例如假冒高階主管、IT 部門或健保署)
- `Financial Request` (財務要求：要求購買點數、匯款、更改薪資入帳帳戶)
- `Curiosity/Greed` (貪婪誘惑：中獎通知、免費領取)

#### 2. 提示詞工程 (Prompt Engineering)
實作初期，可以使用 OpenAI (gpt-4o-mini) 或 Anthropic (Claude 3 Haiku) 的 API 來快速驗證。必須使用 **Few-Shot Prompting (少樣本提示)** 結合 **Structured Output (結構化輸出)**，確保擴充功能不會因為 LLM 亂講話而崩潰。
**系統提示詞 (System Prompt) 範例設計：**

```plaintext
你是一個進階的 SOC 資安意圖分析引擎。請分析使用者輸入的信件/網頁文本，找出潛藏的社交工程攻擊意圖。

【判斷邏輯】：
不要只依賴網址特徵。請深層分析文本是否包含「製造急迫感」、「冒充權威」、「誘騙提供財務或登入資訊」等意圖。

【輸出格式】：
你必須且只能輸出為 JSON 格式，包含以下欄位：
- "is_phishing" (boolean)
- "risk_level" (string: "High", "Medium", "Low")
- "detected_intents" (list: 從 [Urgency, Authority, Financial, Greed, None] 中挑選)
- "explanation" (string: 用繁體中文向一般使用者解釋為什麼這封信很危險，限 50 字內)

【範例 1】：
輸入："親愛的客戶，您的蝦皮帳戶因異常登入已被限制。請於兩小時內點擊以下連結完成驗證，否則將永久停權。"
輸出：{"is_phishing": true, "risk_level": "High", "detected_intents": ["Urgency", "Authority"], "explanation": "此訊息假冒蝦皮客服（冒充權威），並用『兩小時內停權』製造急迫感，誘騙您點擊未知連結，極可能是釣魚詐騙。"}

```

#### 3. 微調 (Fine-tuning) - 爭取高分的殺手鐧
如果只用 Prompt Engineering，教授可能會覺得技術深度不夠。第 15 週的進度報告要求中，明確包含了「微調腳本 (fine-tuning scripts)」或「資料集整理」。
**如何進行微調：**

1. **模型選擇**：選擇輕量級的開源模型，例如 Llama-3-8B-Instruct、TAIDE (台灣特化模型)，或是專門做分類的 RoBERTa/MobileBERT。
2. **資料集建立 (Dataset Curation)**：這是這堂課最看重的部分之一。團隊需要收集約 500-1000 筆**台灣在地化**的釣魚樣本（例如：假冒郵局包裹未領、假冒交通罰單未繳、假冒 LINE Pay 登入異常）。將這些樣本與正常信件混搭，標註好對應的 Intent 與 Risk Level。
3. **訓練手法**：使用 LoRA (Low-Rank Adaptation) 技術進行參數高效微調 (PEFT)。這能讓一般消費級顯卡（甚至用 Google Colab 免費版）就能在幾小時內訓練完畢。微調的目的是讓這個 8B 的小模型，具備與 GPT-4 匹敵的「繁體中文釣魚意圖辨識能力」，同時大幅降低 API 成本與延遲。

### 第三步：團隊分工與期末評估準備
為了滿足第 15 週與第 16 週的整合與評估要求，三人分工可以這樣安排：

- **組員 A (前端與整合)**：負責 Extension 開發 (Content Script DOM 擷取、攔截畫面 UI 實作) 與 FastAPI 後端架設。確保整個系統延遲極低，不會影響使用者平常上網的體驗。
- **組員 B (資料與微調)**：負責爬蟲或收集釣魚資料集，撰寫 Python 腳本清洗資料並轉換為微調所需的格式 (JSONL)。負責在 Colab 上跑 LoRA 微調腳本。
- **組員 C (提示詞工程與評估)**：負責迭代設計 Prompt，並建立測試框架。用保留的 200 筆測試資料集跑出**精確度 (Precision)、召回率 (Recall) 與混淆矩陣 (Confusion Matrix)**。
你們只要在第 15 週報告中秀出：「經過我們使用 800 筆台灣在地化詐騙資料進行 LoRA 微調後，意圖分析的 F1-Score 從基礎模型的 65% 提升到了 92%，且系統反應時間縮短至 0.8 秒」，這樣的科學評估數據，幾乎就能穩拿最終報告的 Outstanding 評級了。
